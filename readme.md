## ログレスの通信プロトコルをデコードするプログラム

ログレスが通信でやり取りデータ（バイト列）を、人間が理解できる形に変換するプログラムです。
ログレスは独自形式の通信プロトコルを使っています。

### そもそも何をしているの？

ログレスはサーバーと通信しながら動いていますが、通信の中身はそのままだと「ただの数字の列（バイト列）」です。
このプログラムではそれを解析して、

- 「これはチャットの内容」
- 「これは敵の位置」
- 「これは自分の装備」

といった意味のあるデータに変換します。

### 何ができるようになるの？

このプログラムを使うと、プログラムから以下のような情報を取得できます。

- 自分宛のチャット内容
- 敵の位置情報
- 自分の持っている武器
- その他ゲーム内部の様々な情報

## パケットを取得する

まずは「ゲームが送受信している通信データ（パケット）」を取得します。
ここではtsharkを使います。(scapyでもOK)

tsharkはWiresharkに付属しているツールです。
コマンドラインからパケットを取得できます。

### ① インターフェース番号を調べる

コマンドラインで以下のコマンドを実行。

```
tshark -D
```

もしtsharkにパスが通っていない場合は、以下のようにフルパスで実行します。

```
"C:\Program Files\Wireshark\tshark.exe" -D
```

出力例

```
1. \Device\NPF_{...} (ローカル エリア接続* 8)
2. \Device\NPF_{...} (ローカル エリア接続* 7)
3. \Device\NPF_{...} (ローカル エリア接続* 6)
4. \Device\NPF_{...} (Bluetooth ネットワーク接続)
5. \Device\NPF_{...} (イーサネット)
```

通常は以下のどちらかを選びます

- Wi-Fi（無線）
- イーサネット（有線）

なので、この出力例だとインターフェース番号は5を使うのがいいです。

### ② パケットを取得する

まず出力用フォルダとしてoutというフォルダを作成しておきます。

```
mkdir out
```

次に以下のコマンドを実行します。

```
tshark -i インターフェース番号 -f "tcp port 8800" -w out/dump.pcap
```

#### 各オプションの意味

- -i：監視するインターフェース番号
- -f：フィルタ（ログレスの通信ポートである8800のみ取得）
- -w：ファイルに保存

#### 実行後の流れ

- コマンドを実行
- ログレスを起動
- 適当に移動したりチャットする
- 十分データが取れたらCtrl + Cで停止

するとout/dump.pcapが作成されます。

### ペイロード（中身）を抽出する

取得したパケットには様々な情報が含まれていますが、今回必要なのはTCPのペイロード（実際のデータ部分）だけです。

これを取り出すために、以下のスクリプトを使います。

```
pcap_to_tcp.py
```

まずscapyをインストールしてください。

```
pip install scapy
```

実行

```
python pcap_to_tcp.py
```

成功するとout/tcp_raw.csvが生成されます。

## デコード（通信データの解析）

ここでは、先ほど取得した「生のバイト列」を解析して、意味のあるデータに変換します。

### ① 必要なライブラリをインストール

この処理では `pandas` を使います。

```bash
pip install pandas
```

### ② デコードを実行

以下のコマンドを実行してください。

```bash
python packet_parser.py out/tcp_raw.csv
```

### ③ 実行結果

成功すると、以下のファイルが生成されます。

```
out/parsed_output.csv
```

## 出力データの見方

このCSVには、通信内容が解析された状態で入っています。

### direction（通信の向き）

通信がどちら向きかを表します。

- `SEND`：クライアント → サーバー（自分が送った）
- `RECV`：サーバー → クライアント（受け取った）

### func_name（通信の種類）

通信の意味を表します。

例：

- `C_GMCL_CHAR_MOVE_REQ`
  → プレイヤーの移動リクエスト

- `C_GMCL_CHAT_SEND_REQ`
  → チャット送信リクエスト

名前を見るだけで何をしている通信かがだいたい分かります。

### 時系列について

出力は時系列順にはなっていません。SENDとRECVに分けてまとめているためです。時系列で見たい場合は、自分で並び替える必要があります。

## 出力例①：移動リクエスト

`C_GMCL_CHAR_MOVE_REQ` の例です（一部抜粋）。

```json
[
    {
        "velocity": {
            "float": 0.0
        }
    },
    {
        "areaUID": {
            "t_AreaUID": [{ "int": 151846954 }, { "int": 1774425063 }]
        }
    },
    {
        "start": {
            "t_MapPos": [
                { "float": 5195.82080078125 },
                { "float": -6004.75048828125 }
            ]
        }
    },
    {
        "end": {
            "t_MapPos": [
                { "float": 4573.56640625 },
                { "float": -5880.2998046875 }
            ]
        }
    }
]
```

ここで分かること

- `start`：移動開始位置
- `end`：移動先

つまり、
どこからどこへ移動したかが分かります

## 出力例②：チャット受信

`S_GMCL_CHAT` の例です。

```json
{
  "senderName": {
    "string": "冒険者"
  },
  "message": {
    "string": "{""type"":""message"",""text"":""テスト""}"
  }
}
```

ここで分かること

- `senderName`：送信者
- `message`：チャット内容

つまり、

誰が何を発言したかが分かります

### チャットの種類（destination_t）

チャットの送信先は `destination_t` で判別できます。

| 値  | 種類     |
| --- | -------- |
| 1   | DM       |
| 8   | エリア   |
| 9   | パーティ |
| 16  | クラン   |
| 32  | グループ |
| 200 | ワールド |
